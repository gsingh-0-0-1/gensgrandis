It's definitely going to be a common question - why develop this game?

Well, I've had the idea for quite some time - since I was around 11 or 12. But I didn't pick up the necessary skills to implement a functional, usable version until recently.

In addition, I wanted to create the game in a way that would prevent extreme centralization and allow users to have more control over the game's functionality.

Thus, I've chosen to put the majority of the game handling code is client-side, which allows the game's servers to host a much larger number of clients than it would be able to otherwise.

However, the unique part about this is that the client-side nature of the game logic won't actually compromise multiplayer gameplay - using the server as a middle-man, game clients can communicate and verify game data by themselves, without the need of the server.

This, again, allows the game to be served to a large number of clients while keeping server-side load relatively low.

The most important part, though, is that this serves as a demonstration that games of decent complexity can be served to clients without them having to do anything - you don't have to download a file or install dependencies; you don't have to care about what OS you're using; you just need to type gensgrandis.com into your browser.

And above all, the game's logic is all available to clients - if you want to modify the game in any way - adding units, modiyfing bonuses - all you have to do is write that code, host it somewhere, and load it in through your browser's console. And you can do this both with singleplayer and multiplayer, all without compromising the integrity of the game's data!
